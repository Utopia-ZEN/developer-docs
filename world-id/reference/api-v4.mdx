<Note>
  Use this endpoint to verify **World ID 3.0 (Legacy)** or **World ID 4.0**
  proofs. Should be used with [IDKit 4.0+](/world-id/id/idkit-v4-preview).
</Note>

```
POST https://developer.world.org/api/v4/verify/{rp_id}
```

Accepts both `rp_id` (`rp_xxx`) and `app_id` (`app_xxx`) as the route parameter.

The `result` from a successful `pollUntilCompletion()` response can be forwarded directly - no transformation needed.

During the preview period, users will produce **v3 proofs** since World ID v4 has not launched yet. The endpoint verifies either format transparently.

### Request Body

<ParamField body="protocol_version" type="string" required>
  The protocol version. Must be `"3.0"` or `"4.0"`. IDKit sets this
  automatically.
</ParamField>

<ParamField body="nonce" type="string" required>
  The nonce used in the RP signature, as returned by IDKit.
</ParamField>

<ParamField body="action" type="string" required>
  The action identifier. Must match the action passed to `IDKit.request()`.
</ParamField>

<ParamField body="action_description" type="string" default='""'>
  Human-readable description of the action. Used when creating the action for
  the first time.
</ParamField>

<ParamField body="responses" type="array" required>
  Array of proof response items (at least one). The shape of each item depends
  on `protocol_version`.
</ParamField>

#### Response item fields (protocol 3.0)

<ParamField body="responses[].identifier" type="string" required>
  Credential type (e.g. `"orb"`).
</ParamField>

<ParamField body="responses[].proof" type="string" required>
  ABI-encoded zero-knowledge proof (hex string).
</ParamField>

<ParamField body="responses[].merkle_root" type="string" required>
  Merkle root hash (hex string).
</ParamField>

<ParamField body="responses[].nullifier" type="string" required>
  Nullifier hash (hex string, optional `0x` prefix).
</ParamField>

<ParamField
  body="responses[].signal_hash"
  type="string"
  default="keccak256('')"
>
  Hash of the signal. Defaults to the keccak256 hash of an empty string.
</ParamField>

<ParamField body="responses[].max_age" type="number">
  Maximum age of the Merkle root in seconds. Range: 3600 (1 hour) to 604800 (7
  days).
</ParamField>

#### Response item fields (protocol 4.0)

<ParamField body="responses[].identifier" type="string" required>
  Credential type (e.g. `"orb"`).
</ParamField>

<ParamField body="responses[].proof" type="string[5]" required>
  Array of exactly 5 hex strings: 4 compressed Groth16 proof elements + Merkle
  root.
</ParamField>

<ParamField body="responses[].nullifier" type="string" required>
  RP-scoped nullifier (hex string, optional `0x` prefix).
</ParamField>

<ParamField body="responses[].issuer_schema_id" type="number" required>
  Credential issuer schema ID.
</ParamField>

<ParamField body="responses[].expires_at_min" type="number" required>
  Minimum credential expiration timestamp (unix seconds).
</ParamField>

<ParamField body="responses[].signal_hash" type="string" default="0x0">
  Hash of the signal. Defaults to `"0x0"`.
</ParamField>

```javascript JavaScript
const completion = await request.pollUntilCompletion();

if (!completion.success) {
  console.error("Verification failed:", completion.error);
  return;
}

const response = await fetch(
  `https://developer.world.org/api/v4/verify/${RP_ID}`,
  {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(completion.result), // forward directly
  },
);
```

</CodeGroup>

### Possible Responses

- `200 OK` - At least one proof was successfully verified.
- `400 Bad Request` - All proofs failed verification, or the user has already verified for this action.
- `404 Not Found` - App not found or no longer active.

### Response Fields

<ParamField body="success" type="boolean">
  Whether at least one proof was verified successfully.
</ParamField>

<ParamField body="action" type="string">
  The action identifier (returned on success).
</ParamField>

<ParamField body="nullifier" type="string">
  The nullifier from the first successful proof (returned on success). Use this
  to track unique users per action.
</ParamField>

<ParamField body="created_at" type="string">
  ISO 8601 timestamp of when the nullifier was recorded.
</ParamField>

<ParamField body="environment" type="string">
  `"production"` or `"staging"`. In staging, nullifier reuse is allowed.
</ParamField>

<ParamField body="results" type="array">
  Per-response verification results. Each entry includes `identifier`,
  `success`, and optionally `nullifier` (on success) or `code`/`detail` (on
  failure).
</ParamField>
